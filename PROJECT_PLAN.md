# 强化学习期末项目完整规划

## 1. 项目概述

### 1.1 项目方向

本项目选择课程要求中的 **Track A: Implementation Track**，题目建议定为：

**基于 Q-Learning 的机器人配送最短路径学习**

如果想把项目做得更有深度，最终可以升级为：

**基于 Q-Learning 与 SARSA 的机器人配送路径学习与对比研究**

### 1.2 项目核心目标

本项目希望完成一个从基础实现到进阶分析都比较完整的强化学习课程项目。整体目标分为两层：

- 完成一个稳定可运行的 baseline
- 在 baseline 之上加入更有分析价值的扩展实验和方法对比

### 1.3 项目最终交付物

根据课程要求，本项目最终需要交付：

- 10 页正文项目报告
- 可运行的 Python 代码
- 清晰的运行说明
- 课堂展示汇报

## 2. 为什么选这个题目

这个题目适合做课程项目，主要因为它在“容易做出来”和“容易做深”之间平衡得很好。

优势如下：

- 环境建模简单，适合从零实现
- Q-learning 与课程内容直接相关
- 结果很容易可视化，适合汇报展示
- 既可以先做出基础版，也可以自然扩展到更深入的研究问题
- 对报告写作非常友好，容易形成“问题-方法-实验-分析”的完整结构

不建议一开始就选过于复杂的应用场景，比如交通调度、医疗调度或网络路由，因为这些题目虽然看起来更高级，但环境建模成本更高，前期很容易把大量时间花在问题定义上。

## 4. 问题定义

### 4.1 任务描述

我们定义一个机器人配送任务：

- 机器人位于二维网格地图中
- 从起点 `S` 出发
- 目标是到达终点 `G`
- 地图中存在障碍物，部分区域不可通行
- 机器人每一步只能选择上下左右四个动作之一
- 任务目标是学会一条总回报尽可能高的配送路径

这个问题可以自然建模为强化学习中的 MDP。

### 4.2 MDP 建模

#### 状态空间

基础版状态可以定义为机器人当前位置：

`s = (row, col)`

如果后续做进阶版，也可以把地图特征、是否到过某区域、是否完成中间任务等纳入状态，但不建议在 baseline 阶段就做复杂状态设计。

#### 动作空间

四个离散动作：

- `up`
- `down`
- `left`
- `right`

#### 状态转移

- 如果动作合法，机器人移动到新位置
- 如果动作越界或撞上障碍物，则位置不变

#### 奖励函数

基础版建议设计为：

- 每走一步：`-1`
- 非法动作或撞障碍：`-5`
- 到达终点：`+20` 或 `+50`

这个奖励设计的直觉是：

- 尽快到终点
- 少绕路
- 避免无效移动

#### 终止条件

一个 episode 在以下情况下终止：

- 到达目标点
- 超过最大步数限制

建议：

- 小地图最大步数：`100`
- 大地图最大步数：`200`

## 5. Baseline 规划

这一部分是项目最重要的基础。建议先确保这一版能稳定跑通，再考虑扩展。

### 5.1 Baseline 目标

Baseline 的最低成功标准应该是：

- 环境正确运行
- Q-learning 能成功训练
- 学到的策略能大概率到达目标
- 能生成训练曲线和路径图
- 可以作为报告中最基础的实验部分

### 5.2 Baseline 环境设定

建议使用一个简单的静态网格地图。

例如：

- 地图大小：`6x6` 或 `8x8`
- 一个固定起点
- 一个固定终点
- 若干静态障碍物

建议地图难度不要一开始太高。第一张图应该保证最短路径清晰存在，并且障碍数量适中。

### 5.3 Baseline 算法

使用 **表格式 Q-learning**。

Q-learning 更新公式：

`Q(s, a) <- Q(s, a) + alpha * [r + gamma * max_a' Q(s', a') - Q(s, a)]`

动作选择使用 epsilon-greedy：

- 以概率 `epsilon` 随机探索
- 否则选当前 Q 值最大的动作

### 5.4 Baseline 推荐超参数

可以从下面这组初始参数开始：

- `alpha = 0.1`
- `gamma = 0.95`
- `epsilon = 1.0`
- `epsilon_min = 0.05`
- `epsilon_decay = 0.995`
- `episodes = 2000`

如果训练不稳定，可以优先调：

- `epsilon_decay`
- `goal reward`
- `step penalty`

### 5.5 Baseline 代码模块

建议项目结构如下：

```text
RL_final_project/
  PROJECT_PLAN.md
  README.md
  requirements.txt
  src/
    env.py
    agent.py
    train.py
    evaluate.py
    visualize.py
    utils.py
  configs/
    base.yaml
  results/
    figures/
    logs/
    models/
  report/
    report.md
```

各文件职责如下：

#### `src/env.py`

- 定义网格世界环境
- 实现 `reset()`
- 实现 `step(action)`
- 处理边界、障碍物和奖励

#### `src/agent.py`

- 定义 Q-learning 智能体
- 维护 Q-table
- 实现 epsilon-greedy 动作选择
- 实现 Q 值更新

#### `src/train.py`

- 编写训练循环
- 记录 reward、步数、成功率
- 保存训练结果

#### `src/evaluate.py`

- 用无探索策略测试训练好的模型
- 统计平均表现

#### `src/visualize.py`

- 画训练曲线
- 画最终路径图
- 生成实验对比图

### 5.6 Baseline 需要完成的图表

至少应生成以下图表：

- episode reward 曲线
- average reward moving average 曲线
- success rate 曲线
- average steps to goal 曲线
- 最终学习路径可视化图

### 5.7 Baseline 最低实验内容

建议最少完成以下实验：

#### 实验 1：训练收敛实验

目的：

- 验证 Q-learning 是否能学会到达目标

指标：

- reward
- success rate
- average steps

#### 实验 2：简单超参数实验

比较：

- 不同 `epsilon_decay`
- 不同 `goal reward`

目的：

- 观察参数对学习稳定性的影响

### 5.8 Baseline 预期结论

Baseline 最终应该能支持以下结论：

- Q-learning 能在简单静态网格环境中学到有效路径策略
- 合理的奖励函数可以显著加快学习速度
- 探索率衰减会影响训练稳定性与收敛速度

## 6. 进阶规划

进阶部分的目标，不是简单堆代码，而是让项目变成“有研究问题、有对比、有分析”的完整课程项目。

### 6.1 进阶目标

进阶版希望回答的问题包括：

- 在更复杂地图下，Q-learning 还能否稳定工作
- 不同奖励设计对最终路径行为有何影响
- Q-learning 和 SARSA 在此类路径规划任务上是否有表现差异
- 环境难度增加后，表格式方法会出现什么局限

### 6.2 最推荐的进阶方向

推荐按下面优先级扩展。

#### 方向 1：更复杂的环境

可以选择以下一项或两项：

- 随机起点
- 更大地图
- 更高障碍密度
- 不同地形代价

最推荐的是：

- 随机起点
- 不同地形代价

因为这两个扩展既不太难实现，又能明显增强分析价值。

#### 方向 2： 在复杂环境里的不同初始化策略

- 用在简单环境里学到的Q-table去初始化在复杂环境的Q-table

- 直接在复杂环境里训练Q-table

  这两种训练策略进行对比，对比不同初始化策略对最终收敛速度，性能的影响



#### 方向 3：算法对比

最推荐加入一个对比算法：

- `SARSA`

理由：

- 和 Q-learning 同样经典
- 实现难度不高
- 非常适合写方法对比分析

如果时间非常充足，还可以考虑：

- `n-step SARSA`
- `Dyna-Q`

但不建议一开始就上更复杂方法。

#### 方向 3：更完整的实验设计

将实验从“能跑”升级为“研究型对比”。

建议围绕以下问题设计实验：

- 简单地图和复杂地图上谁收敛更快
- 不同奖励函数下谁更容易学到短路径
- 不同算法在碰撞风险较高的环境中行为是否不同
- 起点随机时策略是否还能保持稳定表现

## 7. 完整实验矩阵建议

为了兼顾工作量和分析深度，建议实验分层做。

### 7.1 Baseline 实验层

#### 实验 A：基础训练收敛

设置：

- 算法：Q-learning
- 地图：小地图，固定起点终点
- 奖励：标准奖励

输出：

- reward 曲线
- success rate 曲线
- average steps 曲线
- 学习到的最终路径

#### 实验 B：超参数敏感性

比较：

- `epsilon_decay = 0.99 / 0.995 / 0.999`
- `alpha = 0.05 / 0.1 / 0.2`

输出：

- 收敛速度对比图
- 最终性能对比图

### 7.2 进阶实验层

#### 实验 C：奖励函数对比

比较：

- 只有终点奖励
- 终点奖励 + 每步惩罚
- 终点奖励 + 每步惩罚 + 碰撞惩罚

要分析的问题：

- reward shaping 是否帮助学习
- 奖励设计是否影响路径长度与稳定性

#### 实验 D：环境复杂度对比

比较：

- `6x6` 地图
- `8x8` 地图
- `10x10` 地图
- 低障碍密度 vs 高障碍密度

要分析的问题：

- 地图复杂度上升后，学习速度如何变化
- 成功率和步数是否显著恶化

#### 实验 E：随机起点泛化实验

设置：

- 训练和测试中起点随机
- 终点固定

要分析的问题：

- 智能体是否只是记住了一条固定路线
- 策略是否具有一定泛化能力

#### 实验 F：算法对比实验

比较：

- Q-learning
- SARSA

指标：

- success rate
- average reward
- average steps
- convergence speed

要分析的问题：

- off-policy 的 Q-learning 是否更激进
- on-policy 的 SARSA 是否更稳定或更保守

## 13. 风险与应对策略

### 风险 1：Q-learning 学不出来

可能原因：

- 地图太复杂
- 奖励函数设计不合理
- epsilon 衰减过快

应对：

- 先退回小地图
- 增大奖励差异
- 减慢探索率衰减

### 风险 2：结果图不好看，分析不充分

可能原因：

- 只记录了 reward，没有记录成功率和步数
- 缺少路径可视化

应对：

- 补充 success rate 与 average steps
- 一定要画路径图和对比图

### 风险 3：项目看起来像简单作业

可能原因：

- 只有单算法
- 没有对比实验
- 没有研究问题

应对：

- 增加 SARSA 对比
- 增加奖励设计和地图复杂度实验
- 明确写出研究问题

## 15. 最终执行建议

最好的推进顺序不是一开始追求复杂，而是：

1. 先把 baseline 稳定做出来
2. 再让图表和路径结果变得可信
3. 最后再做进阶算法和复杂环境对比

一句话总结就是：

**先做稳，再做深。**

只要 baseline 稳定，你们后面的每一个扩展都会自然变成加分项；如果 baseline 都没站稳，后面的进阶内容反而容易拖垮项目进度。
